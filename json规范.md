这是为您整理的 **JSON 编写核心避坑指南**，涵盖了标准格式与现代配置文件的通用规则：

### 1. 铁律（无论标准 JSON 还是 JSONC 都必须遵守）
*   **双引号法则**：键（Key）和字符串值（String）**必须**使用双引号 `""`。
    *   ❌ 禁止单引号 `'`。
    *   ❌ 禁止键不加引号。
*   **全小写法则**：关键字必须**全小写**。
    *   ✅ 正确：`true`, `false`, `null`
    *   ❌ 错误：`True`, `False`, `None`, `NULL`
*   **Python 隔离**：JSON 不是 Python 字典。
    *   没有 `None`（要用 `null`）。
    *   没有元组 `()`（要用数组 `[]`）。

### 2. 区分场景（关于“逗号”与“注释”）

| 场景 | **标准 JSON** (API数据/传输) | **JSONC** (VS Code配置/本地设置) |
| :--- | :--- | :--- |
| **注释** (`//`) | **❌ 绝对禁止** | **✅ 允许** |
| **尾随逗号** (最后一行) | **❌ 绝对禁止** (会报错) | **✅ 允许** (方便修改) |
| **文件扩展名** | `.json` | `.json` / `.jsonc` |

### 3. 一句话记忆口诀
> **“双引号，全小写；数据传输不留尾（逗号），配置注释看环境。”**

### 4. 最佳实践建议
*   **写数据接口时**：严格遵守标准（无注释、无尾随逗号），这是最安全的。
*   **写配置文件时**：VS Code 等现代编辑器里可以放宽心写注释和逗号，但**语法铁律**（双引号、小写）永远不能破。
